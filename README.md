# Pixel Agent Demo

A demo showcasing 5 Claude Code agents collaborating across the SDLC to build a URL shortener microservice from scratch. Each agent is visualized as an animated pixel character using the [Pixel Agents](https://github.com/pablodelucca/pixel-agents) VS Code extension.

Built for knowledge sharing with VPs and engineers.

## What Happens

Five AI agents work in parallel, each owning a different role:

| Agent | What it does | Output |
|-------|-------------|--------|
| Product Manager | Writes product requirements | `docs/prd.md` |
| Architect | Designs API spec and data model | `docs/architecture/` |
| Developer | Implements the FastAPI service | `src/`, `requirements.txt` |
| QA Engineer | Writes and runs pytest tests | `tests/` |
| DevOps Engineer | Creates Dockerfile, Makefile, CI | `Dockerfile`, `Makefile`, `.github/` |

Agents coordinate through files — downstream agents poll for upstream artifacts before starting. The result is a working URL shortener with three endpoints:

- `POST /shorten` — create a short URL
- `GET /:code` — redirect to the original URL
- `GET /:code/stats` — get click count

## Prerequisites

- **macOS**
- **VS Code** with the [Pixel Agents extension](https://marketplace.visualstudio.com/items?itemName=pablodelucca.pixel-agents)
- **Claude Code CLI** installed and authenticated

## Running the Demo

### 1. Open the project in VS Code

```bash
code pixel-agent-demo
```

### 2. Open the Pixel Agents panel

Command Palette (`Cmd+Shift+P`) → **"Pixel Agents: Show Panel"**

### 3. Run the orchestration script

Open a VS Code terminal and run:

```bash
bash scripts/run-agents.sh
```

### 4. Launch each agent

The script walks you through 5 agents one at a time:

1. Click **"+ Agent"** in the Pixel Agents panel (this opens a Claude session)
2. The role prompt is **automatically copied to your clipboard**
3. **Cmd+V** to paste the prompt into the Claude session, then **Enter**
4. Press **Enter** in the script terminal to move to the next agent

Repeat for all 5 agents. They all run in parallel — the PM starts immediately, and the others wait (polling every 5 seconds) until their upstream files appear.

### 5. Watch the demo

All 5 pixel characters appear in the virtual office. You'll see them working as:

```
PM → Architect → Developer → QA Engineer
                           → DevOps Engineer
```

## Resetting for a New Demo

Remove all agent-generated files:

```bash
bash scripts/run-agents.sh prune
```

This cleans `docs/`, `src/`, `tests/`, `Dockerfile`, `Makefile`, `requirements.txt`, `.github/`, and `*.db` files.
